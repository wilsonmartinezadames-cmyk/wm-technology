# 💳 Guía de Integración: Pagos con Stripe\n\n## Descripción General\n\nEsta guía cubre la integración completa de Stripe para procesar pagos de suscripción en WM TECHNOLOGY.\n\n---\n\n## Configuración de Stripe\n\n### 1. Crear Cuenta Stripe\n\n1. Ve a [https://stripe.com](https://stripe.com)\n2. Crea una cuenta o inicia sesión\n3. Ve a Dashboard → API Keys\n4. Copia tus claves:\n   - **Secret Key** (sk_test_...)\n   - **Publishable Key** (pk_test_...)\n   - **Webhook Secret** (whsec_...)\n\n### 2. Configurar Variables de Entorno\n\nEn `backend/.env`:\n\n```bash\nSTRIPE_SECRET_KEY=sk_test_YOUR_SECRET_KEY_HERE\nSTRIPE_PUBLISH_KEY=pk_test_YOUR_PUBLISHABLE_KEY_HERE\nSTRIPE_WEBHOOK_SECRET=whsec_test_YOUR_WEBHOOK_SECRET_HERE\n```\n\n### 3. Instalar SDK de Stripe\n\n```bash\ncd backend\nnpm install stripe\n```\n\n---\n\n## Endpoints de Pagos\n\n### POST /api/payments/checkout\n\nInicia un checkout de Stripe para una suscripción.\n\n**Request:**\n```json\n{\n  \"plan\": \"pro\"\n}\n```\n\n**Planes disponibles:**\n- `starter` - $29/mes (3 modelos IA, 1k requests)\n- `pro` - $79/mes (10 modelos IA, 10k requests)\n- `enterprise` - $299/mes (unlimited)\n\n**Response (200):**\n```json\n{\n  \"success\": true,\n  \"url\": \"https://checkout.stripe.com/pay/cs_test_...\",\n  \"sessionId\": \"cs_test_...\"\n}\n```\n\n**Uso en Frontend:**\n```javascript\nconst response = await api.post('/payments/checkout', { plan: 'pro' })\nwindow.location.href = response.data.url // Redirige a Stripe Checkout\n```\n\n### GET /api/payments/invoices\n\nObtiene el historial de facturas del usuario.\n\n**Response (200):**\n```json\n{\n  \"success\": true,\n  \"count\": 3,\n  \"invoices\": [\n    {\n      \"_id\": \"64a5f1d9e6b2a1c2d3e4f5g6\",\n      \"userId\": \"...\",\n      \"amount\": 79,\n      \"method\": \"stripe\",\n      \"status\": \"completed\",\n      \"transactionId\": \"ch_1H3fg9...\",\n      \"createdAt\": \"2024-05-25T14:00:00Z\"\n    }\n  ]\n}\n```\n\n### GET /api/payments/subscription\n\nObtiene el estado actual de la suscripción.\n\n**Response (200):**\n```json\n{\n  \"success\": true,\n  \"subscription\": {\n    \"_id\": \"...\",\n    \"plan\": \"pro\",\n    \"status\": \"active\",\n    \"price\": 79,\n    \"startDate\": \"2024-05-25T00:00:00Z\",\n    \"endDate\": \"2024-06-25T00:00:00Z\",\n    \"autoRenewal\": true,\n    \"isActive\": true,\n    \"daysRemaining\": 30\n  }\n}\n```\n\n### PUT /api/payments/subscription\n\nActualiza la suscripción a otro plan.\n\n**Request:**\n```json\n{\n  \"plan\": \"enterprise\"\n}\n```\n\n**Response (200):**\n```json\n{\n  \"success\": true,\n  \"message\": \"Suscripción actualizada\",\n  \"subscription\": { ... }\n}\n```\n\n### POST /api/payments/record\n\nRegistra un pago manualmente (para pruebas).\n\n**Request:**\n```json\n{\n  \"amount\": 79,\n  \"method\": \"stripe\",\n  \"description\": \"Pro Plan - Monthly\"\n}\n```\n\n**Response (201):**\n```json\n{\n  \"success\": true,\n  \"message\": \"Pago registrado\",\n  \"payment\": { ... }\n}\n```\n\n### POST /api/payments/webhook\n\nWebhook para recibir eventos de Stripe.\n\n**Eventos manejados:**\n- `checkout.session.completed` - Pago exitoso\n- `customer.subscription.updated` - Suscripción actualizada\n- `customer.subscription.deleted` - Suscripción cancelada\n\n---\n\n## Webhooks de Stripe\n\n### Configurar Webhook Local\n\n1. **Instalar Stripe CLI:**\n   ```bash\n   # macOS\n   brew install stripe/stripe-cli/stripe\n   \n   # Windows (Chocolatey)\n   choco install stripe\n   ```\n\n2. **Autenticar Stripe CLI:**\n   ```bash\n   stripe login\n   ```\n\n3. **Reenviar eventos a local:**\n   ```bash\n   stripe listen --forward-to localhost:5000/api/payments/webhook\n   ```\n\n4. **Copiar webhook secret:**\n   El CLI mostrará: `whsec_test_...` - cópialo a `.env`\n\n5. **Probar webhook:**\n   ```bash\n   stripe trigger payment_intent.succeeded\n   ```\n\n### Configurar Webhook en Producción\n\n1. Ve a Dashboard → Developers → Webhooks\n2. Haz clic en \"Add endpoint\"\n3. URL del endpoint: `https://tu-dominio.com/api/payments/webhook`\n4. Eventos a escuchar:\n   - `checkout.session.completed`\n   - `customer.subscription.updated`\n   - `customer.subscription.deleted`\n5. Copia el \"Signing secret\" y añádelo a `.env`\n\n---\n\n## Flujo de Checkout\n\n### 1. Usuario selecciona plan en Pricing\n\n```javascript\n// frontend/src/pages/Pricing.jsx\nconst handleCheckout = async (plan) => {\n  try {\n    const response = await api.post('/payments/checkout', { plan })\n    window.location.href = response.data.url\n  } catch (error) {\n    console.error('Error:', error.message)\n  }\n}\n```\n\n### 2. Stripe Checkout\n\n- Usuario completa pago en Stripe\n- Redirige a success URL con session ID\n- o redirige a cancel URL si cancela\n\n### 3. Success Page\n\n```javascript\n// frontend/src/pages/Success.jsx\nuseEffect(() => {\n  const verifyPayment = async () => {\n    const sessionId = new URLSearchParams(location.search).get('session_id')\n    // Verificar pago completado\n    // Actualizar perfil del usuario\n  }\n  verifyPayment()\n}, [])\n```\n\n### 4. Webhook actualiza BD\n\n- Evento `checkout.session.completed`\n- Actualizar status de Subscription a \"active\"\n- Enviar email de confirmación\n\n---\n\n## Testing\n\n### Tarjetas de Prueba Stripe\n\n| Tipo | Número | CVC | Fecha |\n|------|--------|-----|-------|\n| Visa Éxito | 4242 4242 4242 4242 | 123 | 12/25 |\n| Visa Rechazada | 4000 0000 0000 0002 | 123 | 12/25 |\n| Visa Decline | 4000 0000 0000 0069 | 123 | 12/25 |\n\n### Testing con cURL\n\n```bash\n# 1. \"stripe\", \"paypal\", \"cash\"\n  status: String,         // \"pending\", \"completed\", \"failed\", \"refunded\"\n  transactionId: String,  // ID de Stripe\n  metadata: Object,       // Datos adicionales\n  createdAt: Date,\n  updatedAt: Date\n}\n```\n\n### Subscription\n\n```javascript\n{\n  userId: ObjectId,\n  plan: String,           // \"starter\", \"pro\", \"enterprise\"\n  status: String,         // \"active\", \"canceled\", \"past_due\"\n  price: Number,          // En dólares/mes\n  startDate: Date,\n  endDate: Date,\n  stripeSubscriptionId: String,\n  autoRenewal: Boolean,\n  features: Object,       // Características del plan\n  createdAt: Date,\n  updatedAt: Date\n}\n```\n\n---\n\n## Seguridad\n\n✅ **Verificación de Webhook:** Firma de Stripe validada\n✅ **PCI Compliance:** Stripe maneja datos sensibles\n✅ **Tokens JWT:** Rutas protegidas requieren autenticación\n✅ **Rate Limiting:** Recomendar implementar en producción\n✅ **HTTPS:** Obligatorio para Stripe\n\n---\n\n## Troubleshooting\n\n### Error: \"Webhook signature verification failed\"\n\n→ Verifica que `STRIPE_WEBHOOK_SECRET` sea correcto en `.env`\n→ Asegúrate de que el Secret Key también sea válido\n\n### Error: \"Invalid API Key\"\n\n→ Verifica que `STRIPE_SECRET_KEY` comience con `sk_test_` o `sk_live_`\n→ Copia de nuevo desde Dashboard → API Keys\n\n### Webhook no recibe eventos\n\n→ Ejecuta: `stripe listen --forward-to localhost:5000/api/payments/webhook`\n→ Copia el `whsec_test_...` a `.env`\n\n### Tarjeta rechazada en checkout\n\n→ Usa tarjeta de prueba válida (4242 4242 4242 4242)\n→ Verifica que el plan sea válido (\"starter\", \"pro\", \"enterprise\")\n\n---\n\n## Próximas Mejoras\n\n- [ ] Integración PayPal\n- [ ] Email de confirmación después de
